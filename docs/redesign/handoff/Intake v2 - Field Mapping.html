Shim label mismatch — fix before ship

The shim writes short goal labels into services_needed, but the live GOAL_SERVICE_TAGS map is keyed by the old long labels — so the lookup misses and provider / mentor / lead matching falls back to location-only. The SQL backfill has the inverse problem (keyed on long labels, never fires on v2 rows). Fix: add a legacy_label to each GoalDef, or re-key GOAL_SERVICE_TAGS by goal_id and ship that one-line edge change.

Streaming progress needs new instrumentation

The "Building your report" UI is honest about phases, but the pipeline only console.logs them — the sole client-visible state is status (pending → processing → completed/failed). To wire real streaming, add a progress jsonb column (or a generation-phase events table) the client can poll or subscribe to. The new intake_form_events table covers intake funnel analytics, not generation phases.
Notes. "Legacy (shim)" shows how mapV2ToLegacyIntake() projects each v2 field into the flat shape generate-report already consumes, so the redesign can ship without rewriting the pipeline (subject to the goal-label fix above). New columns are additive — the draft migration keeps every legacy column intact and backfills history.
